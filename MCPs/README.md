# MCPs

Generated MCP servers — one directory per learned web application.

Each subdirectory contains a complete, ready-to-use MCP server that was auto-generated by the `/learn-webapp` skill.

## Structure

```
MCPs/
└── <app-name>/
    ├── server/           # The MCP server package
    │   ├── index.mjs     # Server entry point (tool registrations + CDP connection)
    │   ├── commands.mjs  # Learned JavaScript functions (executed via page.evaluate)
    │   ├── manifest.json # Capabilities metadata
    │   └── package.json  # Dependencies (@modelcontextprotocol/sdk, puppeteer-core)
    └── exploration/      # Exploration artifacts from the learning phase
        ├── log.json      # Exploration log (actions taken, elements found)
        └── snapshots/    # Accessibility tree snapshots captured during exploration
```

## How Servers Work

Each server connects to Chrome via CDP (Chrome DevTools Protocol) and executes learned JavaScript functions directly in the page context. No screenshots, no coordinate math — just semantic function calls.

The server is registered in the project's `.mcp.json` and communicates over stdio using the Model Context Protocol.
