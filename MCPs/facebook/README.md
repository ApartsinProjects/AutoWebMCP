## Obfuscated DOM

Facebook uses compiled/hashed CSS class names (e.g., `x9f619`, `xh8yej3`) that change
on every deployment. All selectors use ARIA attributes (`aria-label`, `role`) and text
content matching — CSS class selectors are not used.

## Installation

1. Clone the repo and navigate to the server directory:

   ```bash
   cd MCPs/facebook/server
   npm install
   ```

2. Start Chrome with remote debugging enabled:

   ```bash
   # macOS
   /Applications/Google\ Chrome.app/Contents/MacOS/Google\ Chrome \
     --remote-debugging-port=9222 \
     --user-data-dir="$HOME/chrome-cdp-profile"

   # Windows (Git Bash)
   "$LOCALAPPDATA/Google/Chrome/Application/chrome.exe" \
     --remote-debugging-port=9222 \
     --user-data-dir="$LOCALAPPDATA/Google/Chrome/CDP-Profile"

   # Linux
   google-chrome --remote-debugging-port=9222 \
     --user-data-dir="$HOME/.chrome-cdp-profile"
   ```

3. Log into Facebook in the CDP Chrome window (first time only).

4. Add to your Claude Code MCP config (`.mcp.json`):

   ```json
   {
     "mcpServers": {
       "facebook": {
         "type": "stdio",
         "command": "node",
         "args": ["MCPs/facebook/server/index.mjs"],
         "env": {
           "CHROME_CDP_URL": "http://127.0.0.1:9222",
           "BROWSER_MODE": "visible",
           "DATA_MODE": "user"
         }
       }
     }
   }
   ```

5. Restart Claude Code. The MCP tools are now available.

## Mode Switches

| BROWSER_MODE | DATA_MODE | Use Case |
|---|---|---|
| `visible` | `user` | **Default.** Works in user's real browser |
| `visible` | `sandbox` | Testing — visible browser, clean profile |
| `headless` | `user` | Background automation with user credentials |
| `headless` | `sandbox` | CI/testing — fully isolated |
